About

I am a senior at Knox College with a major in Computer Science and a minor in Mathematics. I drifted through the Computer Science program for a while working with Java and C before stumbling upon web development. I strongly enjoy working with both HTML/CSS and Javascript. They have brought back the fun of development for me. I have expanded my base with more back-end technologies like node.js, Express, and MongoDB. I find the most fun in solving problems even if sometimes the little things are the most frustrating. Web development is a place where I feel like creativity and problem solving come together for a fulfilling experience.

When not coding or designing, I enjoy watching sports and playing complicated strategy games. If it doesn't take you 100 hours of failure to get moving in the right direction, what's the fun of a game?

Skills

I enjoy working with HTML5 and CSS3 everyday. With HTML it's about being better organized and designing things better so that my CSS is better. With CSS (and recently exploring SASS), it's more about learning how little I know. Every day I find some cool new concept that makes pages look cleaner, more professional, or smoother. When I look back through my work, I can see clear places where I learned a new skill (and then overused it everywhere). Experience is had with Bootstrap and it jumps in and out of the code that I use. However, I tend to use Bootstrap as a guide to how things should look, and then go back and code it myself (sometimes it looks rougher, but I definitely understand it better).

Currently the language I enjoy working with the most is Javascript because of how flexible and fluid it is compared to something like Java or C that I came from following a traditional college path. While a lot of my Javascript is displayed in jQuery or AJAX for DOM manipulation or server calls, building even half finished projects in pure Javascript is enjoyable. I love delving into a language and figuring out how to get the most out of it.

While my interest is in front-end development, I've also had to explore the back-end side to get websites up and running. I used node.js with Express and while I have a little experience with PHP and MySQL, most of my serious websites have used MongoDB and Mongoose to store and retrieve data. While working with back-end I started using Jade and while it does make the code look cleaner and cuts down on a lot of repetition, most of the time it's simple HTML that I use on the front-end.
